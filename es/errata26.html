Todas las Arquitecturas

  • 024: PARCHE DE SEGURIDAD: 26 de mayo de 2000
    El núcleo contenía una llamada del sistema no documentada para bloquear operaciones de semáforo mientras se probaban con la orden ipcs(1). Este bloqueo se podría usar como un ataque de denegación de servicio local que bloquearía la salida a los procesos que tuvieran ubicados recursos de semáforo. Los procesos que no usen semáforos no les afecta, así pues el efecto real es mínimo.
    Está disponible un parche gigante que soluciona este problema.

  • 023: PARCHE DE SEGURIDAD: 25 de mayo de 2000
    Una mala utilización de las reglas de ipf(8) keep-state pueden dar como resultado que las reglas del cortafuegos se ignoren. Este parche también incluye soluciones para un cuño de tiempo no alineado, y soluciones de fiabilidad para ipmon y el proxy de ftp de in-kernel.
    Existe un macro parche, que soluciona este problema, y que actualiza ipf a la versión 3.3.16.

  • 022: PARCHE DE SEGURIDAD: 25 de mayo de 2000
    Hay un ataque de localhost contra xlockmore que permite recuperar el resumen criptográfico de la contraseña de root. El daño de este ataque a los sistemas que usan contraseñas DES es considerable, pero en los sistemas con una contraseña de root bien escogida con codificación blowfish (ver crypt(3)) el impacto se reduce bastante.
    (Nota aparte: Nosotros no lo consideramos un agujero de seguridad de root en un localhost en la instalación predefinida, ya que todavía no hemos visto un cracker de blowfish rápido ;-)
    Está disponible un parche para el código que soluciona este problema. Ésta es el segundo parche diseñado para solucionar este problema.

  • 021: RZSZ FISGÓN: 31 de enero de 2000
    El porte rzsz ha sido borrado de la colección de portes, ya que recoge información sobre el usuario y la envía a una dirección de correo previamente designada, espiando de este modo al usuario. Si lo tiene instalado se recomienda que lo elimine de su paquete.

  • 020: MEJORA DE BIBLIOTECA: 26 de enero de 2000
    syslog(3) no reintentaba abrir el "socket", y por tanto el newsyslog(8) nocturno hacía que syslogd(8) no pudiera ver mensajes nuevos. Existe un parche para el código fuente que soluciona este problema.

  • 019: MEJORA DE DISPOSITIVO: 20 de enero de 2000
    Las tarjetas fxp de Intel con PHYs National Semiconductor (nsphy) tienen problemas al negociar y mantener la integridad de enlaces de 100Mb.
    Existe un parche para el código fuente que resuelve este problema.

  • 018: PARCHE DE SEGURIDAD: 20 de enero de 2000
    Los sistemas con procfs activado y montado son vulnerables a un efecto de la salida de stderr de los procesos setuid que son dirigidos a un descriptor preseleccionado en la estructura de almacenamiento de datos de su memoria de procfs.
    Nótese que procfs no se monta por definición en OpenBSD.
    Existe un parche que soluciona este problema.

  • 015: PARCHE EFECTO 2000: 9 de enero de 2000
    La orden at(1) no ha sido capaz de analizar algunos tipos de fechas.
    Existe un parche para el código fuente que soluciona este problema.

  • 013: PARCHE EFECTO 2000: 3 de enero de 2000
    Un pequeño problema con el soporte en el ingreso (login) para la orden adduser(8).
    Existe un parche para el código fuente que soluciona este problema.

  • 012: MEJORA DE DISPOSITIVO: 3 de enero de 2000
    3C900B-TPO falla al seleccionar el tipo de medios (nunca ve ni envía paquetes).
    Existe un parche que soluciona este problema.

  • 011: PARCHE DE SEGURIDAD: 4 de diciembre de 1999
    Varios errores en poll(2) pueden causar una caída en el núcleo.
    Existe un parche para el código fuente que soluciona este problema.

  • 010: PARCHE DE SEGURIDAD: 4 de diciembre de 1999
    Sendmail tiene una condición de carrera en la gestión de los ficheros de alias.
    Existe un parche para el código fuente que soluciona este problema.

  • 009: MEJORA DE DISPOSITIVO: 4 de diciembre de 1999
    Se han hecho varias mejoras al subsistema IDE/ATAPI desde que se lanzó la versión 2.6.
    Algunas de estas mejoras han conseguido que dispositivos testarudos funcionen mucho mejor.
    Está disponible la revisión 1 de este macro parche para el código fuente.

  • 016: PARCHE DE SEGURIDAD: 2 de diciembre de 1999
    Un desbordamiento de la memoria intermedia en el código de RSAREF que se incluye en la versión para los EE.UU. del paquete libssl (llamado sslUSA), se puede explotar probablemente en isakmpd si se usan o activan las características SSL/RSA.
    OpenSSH y httpd (con -DSSL) no son vulnerables a este problema de seguridad.
    NOTA: Los usuarios internacionales que utilicen el paquete ssl26 no están afectados por este problema de seguridad.

    Para comprobar de qué paquete dispone, escriba 

    # pkg_info sslUSA26
    La biblioteca parcheada dice:
    "ssl26.1 USA-only non-commercial crypto libs incl. SSL & RSA"

    Los usuarios de la versión no comercial en EE.UU. que instalaron el paquete antes del 3 de diciembre, deben actualizar su paquete sslUSA26 del siguiente modo:

    # pkg_delete sslUSA26
# pkg_add -v sslUSA26.tar.gz
    usando los nuevos ficheros sslUSA26.tar.gz que han sido puestos a disposición en las réplicas de FTP.
    Para más información lea el aviso de seguridad.
    NOTA: este problema ha resultado ser potencialmente explotable en OpenSSH.

  • 017: FUNCIONALIDAD AÑADIDA: 14 de noviembre de 1999
    Fortran no funciona correctamente. Falta el fichero /usr/include/g2c.h en la versión.
    Existe un parche para el código fuente que soluciona este problema.
    El parche arregla el árbol de fuentes y describe cómo añadir correctamente el fichero include a su sistema.

  • 005: FUNCIONALIDAD AÑADIDA: 11 de noviembre de 1999
    Se han hecho varias mejoras a OpenSSH desde que se lanzó la versión 2.6.
    Para añadir las características (no relacionadas con la seguridad) que desee el usuario, estamos haciendo un macro parche. Ésta ya es la VERSIÓN CUATRO.
    Se encuentra disponible la revisión 4 de este macro parche para el código fuente.
    NOTA: /etc/sshd_config y /etc/ssh_config pueden necesitar algunos cambios.

  • 003: PARCHE DE FUNCIONALIDAD: 10 de noviembre de 1999
    m4 está bastante estropeado en la versión 2.6.
    Se encuentra disponible la versión 3 del parche para el código fuente que soluciona este problema.

  • 002: PARCHE DE SEGURIDAD: 9 de noviembre de 1999
    Cualquier usuario puede cambiar las configuraciones de la interfaz de medios.
    Existe un parche para el código fuente que soluciona este problema.

  • 001: PARCHE DE FIABILIDAD: 8 de noviembre de 1999
    Una condición de carrera en newsyslog(8) puede causar errores in la rotación del fichero de log.
    Un parche para el código fuente soluciona este problema.
