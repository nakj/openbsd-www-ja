Todas las Arquitecturas

  • PARCHE DE SEGURIDAD: 30 de agosto de 1999
    Asegúrese de que arg[] en cron(8) termina con NULL en el popen falso, y que ejecuta sendmail como usuario, no como root. Existe un parche que soluciona este problema.

  • PARCHE DE SEGURIDAD: 12 de agosto de 1999
    Los sistemas de archivos procfs y fdescfs sobrepasan el límite del tiempo al manejar uio_offset en sus rutinas readdir() (estos sistemas de archivos no están activados por definición). Existe un parche que soluciona este problema.

  • PARCHE DE SEGURIDAD: 9 de agosto de 1999
    Al usar execve() en un nuevo proceso se para la perfilación (ver profil(2)). Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD: 6 de agosto de 1999
    Los paquetes que deberían ser manejados por IPsec pueden ser transmitidos como texto en claro. Las fechas de caducidad de PF_KEY SA pueden provocar fugas en los recursos del núcleo del sistema. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD: 5 de agosto de 1999
    Use mktemp(1) en /etc/rc para reescribir motd, y cambie el find(1) para que use -execdir. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD: 30 de julio de 1999
    No permita que usuarios normales usen chflags(2) o fchflags(2) en dispositivos de caracter o bloque de los que puedan ser propietarios. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD: 27 de julio de 1999
    Haga que groff(1) sea invocado con el indicador -S cuando sea llamado por nroff(1), para evitar varias funciones de groff que pueden representar problemas de seguridad. En general, esto no es un problema de seguridad pero ha sido expuesto en BUGTRAQ como si lo fuera. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FIABILIDAD: 19 de mayo de 1999
    Los programas que usan fts(3) podrían provocar un volcado de memoria cuando se les pasa una estructura de directorio con un gran número de entradas. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FIABILIDAD: 19 de mayo de 1999
    Los números de secuencias pueden envolver con TCP_SACK y TCP_NEWRENO, dando como resultado un fallo en la retransmisión. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FIABILIDAD
    Los paquetes TCP retransmitidos podrían corromperse al pasar por un túnel ESP IPSEC. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FIABILIDAD
    Un usuario local puede bloquear el sistema leyendo un archivo de más de 68MB desde una partición ext2fs. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FIABILIDAD
    Las operaciones de socket PF_KEY provocan fugas internas en los recursos del núcleo del sistema, de modo que un sistema que esté ejecutando un dæmon de gestión de claves de IPsec, como photurisd o isakmpd, hará que el subsistema de red pare de funcionar después de una cantidad finita de tiempo. Existe un parche para el código fuente que resuelve este problema.

i386

  • PARCHE EFECTO 2000: 30 de agosto de 1999
    Este parche corrige varios problemas con el año 2000 en OpenBSD/i386 2.5. La versión 2.6 (lanzada el 1 de diciembre de 1999) tiene este problema solucionado. Existe un parche para el código fuente que soluciona este problema.

  • PARCHE DE FIABILIDAD
    Si un usuario abriera el dispositivo brooktree en un sistema en el que no existiera, provocaría una caída del núcleo del sistema. Existe un parche para el código fuente que soluciona este problema.

mac68k

  • PROBLEMA EN LA INSTALACIÓN
