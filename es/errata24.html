Todas las Arquitecturas

  • PARCHE DE FIABILIDAD
    Un usuario local puede provocar una caída del sistema leyendo un archivo de más de 68MB desde una partición ext2fs. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FIABILIDAD
    Un usuario local puede provocar una caída del sistema debido a un error en la capa de vfs. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    El uso de poll(2) puede provocar la caída de la máquina. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    El uso de link(2) en FFS puede provocar la caída de la máquina. Existe un parche para el código fuente que resuelve este problema. Ésta es la versión 4 del parche.

  • PARCHE DE SEGURIDAD
    Existe un desbordamiento de la memoria intermedia en ping(8) que puede contener un problema de seguridad. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    Una condición de carrera en el manejo de ipq por IP permite provocar una caída de forma remota. Existe un parche para el código fuente que resuelve este problema. El parche debe ser instalado después del parche maxqueue y del parche tcp decoding.

  • PARCHE DE SEGURIDAD
    Una condición de carrera entre accept(2) y select(2) permite que un atacante pueda colgar los sockets de forma remota. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    El ensamblaje del fragmento IP puede atascar la máquina en exceso y causar problemas. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FUNCIONALIDAD
    Las llamadas del sistema readv(2) y writev(2) no aceptan un struct iovec con un iov_len de 0. Esto provoca que un test db en perl falle. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    El manejo de RST por TCP/IP estaba demasiado desordenado. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FUNCIONALIDAD
    Durante el arranque, kvm_mkdb puede fallar dando el error "kvm_mkdb: cannot allocate memory". Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FUNCIONALIDAD
    Existe un problema al escribir a los sistemas de archivos montados de la versión 3 de NFS desde un huésped del sistema operativo Solaris 7. Intenta generar ficheros que dan como resultado un error del tipo "Inappropriate file type or format". Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FUNCIONALIDAD
    Existe un problema con los montajes en la versión 3 de NFS en máquinas "big endian" (m68k, sparc y powerpc), al montar sistemas de archivos mayores de 2GB. Se puede ver la evidencia del error ejecutando df(1) y buscando tamaños de particiones negativos. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    Existe un problema de seguridad en las bibliotecas curses y ocurses que afecta a los programas setuid enlazados con -lcurses o -locurses. Existe un parche para el código fuente que resuelve este problema.
    Existen versiones precompiladas de libcurses y libocurses para la plataforma i386. Desarchívelas en /usr/lib.

  • PARCHE DE FUNCIONALIDAD
    xterm hace que vi no restaure la posición correcta del cursor al finalizar. Existe un parche para el código fuente que resuelve este problema.
    De modo alternativo, también puede obtener un fichero terminfo precompilado para instalarlo como /usr/share/misc/terminfo.db.
    Para i386, alpha y mips use la versión little endian.
    Para sparc, m68k y powerpc use la versión big endian.

  • PARCHE DE FUNCIONALIDAD
    El soporte para userdir fue omitido de modo accidental de httpd(8). Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    Existe un problema en bootpd(8) que puede ser un "exploit". bootpd está desactivado por definición en OpenBSD, pero algunos usuarios pueden estar usándolo. Existe un parche para el código fuente que resuelve este problema. Ésta es la versión 2 del parche.

  • PARCHE DE SEGURIDAD
    Existe un problema de bloqueo en una máquina remota en el código de descodificación de TCP. Existe un parche para el código fuente que resuelve este problema.

i386

  • PARCHE DE SEGURIDAD
    Éste es otro parche para una caída del núcleo del sistema provocada por el programa crashme. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FUNCIONALIDAD
    El núcleo del sistema usaba una localización fija en el código para el vector de argumentos pasados por el cargador de arranque /boot. Este problema era la causa de que /boot pudiera introducir el vector de los argumentos de arranque en ninguna otra localización, provocando una caída temprana en el núcleo durante la fase de autoconfiguración. En 2.5, los bloques de arranque se modificarán para usar una nueva localización. Si desea que los viejos núcleos arranquen sobre un nuevo bloque de arranque, tendrá que enlazarlos con este parche. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE SEGURIDAD
    El manejo de la trampa de rastros con la configuración de DDB puede causar una caída del sistema. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE FUNCIONALIDAD
    El arranque de la instalación en i386 tiene un error en la extensión de la señal que evita la inicialización del bloque de arranque correcto cuando el sistema de archivos root se ubica más allá de los 4GB. Existe un parche para el código fuente que resuelve este problema.

    Desafortunadamente, no están disponibles los disquetes de instalación actualizados de 2.4. Asegúrese de que su sistema de archivos root está por debajo de los 4GB de momento.

mac68k

sparc

  • PARCHE DE DISPOSITIVO
    Los dispositivos hme(4) y be(4) de sparc funcionan a duras penas en algunos tipos de máquinas SS-20. Esto se debe a que esas máquinas mienten, diciendo que soportan las explosiones DMA de 64 bits. Ninguna máquina sun4m tiene soporte para esto. Existe un parche para el código fuente que resuelve este problema.

  • PARCHE DE DISPOSITIVO
    El dispositivo le(4) de sparc realiza cambios de medios incorrectos en un tipo de tarjeta bus le(4). Existe un parche para el código fuente que resuelve este problema.
