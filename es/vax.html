OpenBSD/vax

OpenBSD/vax es el porte de OpenBSD para un amplio grupo de máquinas VAX fabricadas por Digital (luego pasó a Compaq y ahora a Hewlett-Packard).

Hay una lista de correo dedicada al porte de OpenBSD/vax en vax@openbsd.org. Para darse de alta en esta lista hay que enviar un mensaje a majordomo@openbsd.org con la orden subscribe vax en el contexto. Por favor, lea nuestras normas sobre las listas de correo antes de suscribirse.

En la actualidad, los mantenedores del porte son Brandon Creighton y Hugh Graham. ¡Cualquiera que quiera contribuir es bienvenido!

Cronología:

El soporte de OpenBSD para la arquitectura VAX data de principios del año 2000, cuando se sincronizó el código fuente con el porte de NetBSD para VAX. Dado que fue la segunda arquitectura sobre la que funcionón BSD Unix (¡en 1979!), es justo que todavía se pueda disponer de un sistema operativo moderno para esas viejas máquinas VAX.

El código base actual proviene en su mayor parte de Berkeley, con contribuciones de código libre de Digital, Mt Xinu, NetBSD, OpenBSD, y otros.

Después de que el porte fuera sincronizado con el código base de NetBSD, se efectuaron reparaciones y mejoras, como el soporte para 16 particiones por dispositivo de disco, soporte para más modelos de VAX, mejora en la emulación de instrucciones no implementadas, y otras.

El desarrollo es continúo, y aún hoy sigue habiendo intercambio de modificaciones entre los desarrolladores de VAX. El resultado de este trabajo es un sistema operativo estable y un entorno familiar para cualquier usuario de OpenBSD.

Estado actual:

En la actualidad, todos los equipos y componentes que aparecen en la lista de hardware con soporte arrancan en modo multiusuario, y hay suficiente soporte para que los dispositivos integrados se puedan utilizar con fines generales. Sin embargo algunos modelos sólo pueden ejecutarse en modo sin disco de momento.

Dado que en este momento no hay soporte para ningún marco de memoria de imagen (frame buffer) en ninguna VAXstation, no hay ningún servidor del X Window System disponible. Sin embargo, existe un juego completo de clientes y utilidades de X que permiten a las máquinas OpenBSD/vax comportarse como servidores de fuentes de X11, o ejecutar clientes de X en sistema remotos.

Todavía quedan unos pocos problemas por resolver en el porte de OpenBSD/vax:

  • El compilador actual tiene a veces problemas con el optimizador (o sea, gcc -O2 produce código incorrecto, mientras que -O1 ó -O2 funcionan). Por lo que sabemos, se están solucionando lentamente. De todos modos la mayor parte del código debería funcionar.
  • Límites en el módulo de pmap. Pueden ser la causa de p´nico en el núcleo. Se han tratado algunas de estas situaciones y falla de forma suave, pero podrían quedar algunos casos sin solucionar.

Proyectos (sin orden):

  • Escribir un controlador para el HDC9224 (controlador de disquete / disco duro MFM) de VAXstation 2000.
  • El soporte para MSCP puede ser algo duro.
  • Escribir controladores de marcos de memoria de imagen.
  • Migrar a ELF y bibliotecas compartidas.
  • Soporte para cualquier sistema nuevo que nos encontremos.

Hardware con soporte:

La lista de hardware con soporte no es exhaustiva. Debido a que el inventorio de sistemas disponible para nuestros desarrolladores, no se han comprobado todas las combinaciones de equipos y componentes.

Como resultado, OpenBSD/vax puede funcionar en una máquina que no esté en esta lista, y necesitamos que los usuarios hagan la prueba y nos envín información para mantener esta lista actualizada. Para ayudarnos a mejorar la exactitud de esta lista, POR FAVOR, envíe la salida de la orden dmesg después de la instalación a dmesg@openbsd.org.

Modelos con soporte

Hay soporte para casi todos los dispositivos integrados en estos modelos, a excepción de los marcos de memoria de imagen y las interfaces de teclado. El soporte para el controlador SCSI sólo existe en donde aparece en la lista.

  • VAXstation/MicroVAX 2000
  • VAXstation 3100/{30,40} (SCSI)
  • VAXstation 3100/{38,48} (SCSI)
  • VAXstation 3100/76 (SCSI)
  • VAXstation 4000/VLC (SCSI)
  • VAXstation 4000/60 (SCSI)
  • VAXstation 4000/90 (SCSI)
  • VAXstation 4000/96 (SCSI)
  • MicroVAX 3100/{10,20}{,e} (SCSI)
  • MicroVAX 3100/{30,40} (SCSI)
  • MicroVAX 3100/80 (SCSI)
  • MicroVAX 3100/90 (SCSI)
  • MicroVAX 3300
  • VAX 4000 100 (SCSI)
  • VAX 4000 105 (SCSI)

Se busca documentación:

  • Información sobre programación de marcos de memoria LCG o SPGFX.
  • Referencias de software para SHAC DSSI.
  • Cualquier edición de VARM.

Cómo obtener e instalar OpenBSD/vax:

La última versión final para OpenBSD/vax es OpenBSD 3.4. Hay unas instrucciones de instalación para OpenBSD/vax 3.4 disponibles.

Hay versiones preliminares (snapshots) disponibles cada cierto periodo de tiempo desde este sitio, así como desde las réplicas de ftp. También hay unas intrucciones de instalación para versiones preliminares de OpenBSD/vax.
