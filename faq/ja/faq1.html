1.1 - OpenBSD って何でしょう ?

OpenBSD プロジェクトは、フリーで入手可能なマルチプラットホームの 4.4BSD ベースの UNIX 系オペレーティングシステムです。 私たちが強調するのは 移植性、標準化、正しさ、 セキュリティです。

この FAQ は、特に最も最近の OpenBSD のリリースであるバージョン 4.9 だけを網羅しています。

1.2 - OpenBSD はどのようなシステム上で動くのでしょう ?

OpenBSD 4.9 は以下のプラットホーム上で動きます。

  • alpha - FTP のみ
  • amd64 - CD に収録
  • armish - FTP のみ
  • hp300 - FTP のみ
  • hppa - FTP のみ
  • i386 - CD に収録
  • landisk - FTP のみ
  • loongson - FTP のみ
  • macppc - CD に収録
  • mvme68k - FTP のみ
  • mvme88k - FTP のみ
  • sgi - FTP のみ
  • socppc - FTP のみ
  • sparc - FTP のみ
  • sparc64 - CD に収録
  • vax - FTP のみ
  • zaurus - FTP のみ

CD に収録 というのはそのプラットフォームが 公式 CD セット に多くのパッケージと ともに収録されているということです。 いくつかのプラットフォーム向けベースシステムCD ISOイメージもまた ダウンロード可能です。

OpenBSD プラットフォームに関する更なる情報は、 プラットフォーム のページをご参照ください。

どうしてそんなにたくさんの "奇妙な"マシンをサポートするのですか、と時々聞かれます。 「我々が欲するからです」が答えです。 もし、十分なスキルを持った人が（時には「十分」が本当にスキルを持った 一人でも！）そのプラットフォームをサポートする事を望むなら、 そのプラットフォームをサポートします。 新しいプラットフォームが現れた時、コードツリーは移植性を破壊するバグや デザインの欠陥から相対的に自由でいられることが、 OpenBSDがマルチプラットフォームを続ける事による実用的な利益です。 OpenBSDプラットフォームは32ビット及び64ビットプロセッサー、リトルエンディアンと ビッグエンディアンのマシン、そしてたくさんの違ったデザインを含んでいます。 そしてもちろん、より「普通の」プラットフォーム向けの質の高いコードベースが 「変な」プラットフォームをサポートする助けになります。

1.3 - なぜ私はOpenBSDを使わなければいけないのでしょうか?

新しいユーザは、OpenBSD が他のフリーな UNIX 系 OS に比べて優れている のかどうか、頻繁に知りたがるものです。しかし、この質問は、おおむね 回答不能ですし、無数の (そして無益な) 宗教論争のネタになってきたものです。 いかなる状況においても、このような質問は OpenBSD のメーリングリストに 投げたりしないようにしてください。

以下に、なぜ私たちが OpenBSD を有用な OS だと思うのか、 いくつか理由をあげてみました。しかし、OpenBSD があなたに とって適切かどうかは、あなた自身が判断するしかありません。

  • OpenBSD はいろいろな ハードウェアプラットホーム上で動きます。
  • OpenBSD は、決して終ることのない徹底的な ソースコードのセキュリティ監査のおかげで、多くのセキュリティ専門家 によって、最も高いセキュリティ を誇る UNIX 系 OS と判断されています。
  • OpenBSD は、ソースコードの形で提供されている、 完全な UNIX 系 OS で、無料です。
  • OpenBSD には、最先端のセキュリティ技術が統合されていますので、 ファイアウォールやプライベート ネットワークサービスを分散環境で構築するのに適しています。
  • OpenBSD は、いろいろな分野での強力な継続的開発の成果の提供を受けて、 できたての技術を国際的なプログラマやエンドユーザのコミュニティとともに 活用する機会を提供しています。
  • OpenBSDはカスタマイズや微調整の必要性を 最小化しようとします。 大多数のユーザーにとっては、 OpenBSD は彼らのハードウェア上で、ソフトウェアが "すぐに動作します"。微調整やカスタマイズはほとんど必要ありません。 それは大変 やる気を失わせてしまいます。

1.4 - OpenBSD って本当にフリーなのでしょうか ?

OpenBSD はすべてフリーです。バイナリはフリーですし、ソースも フリーです。OpenBSD のすべての部分は、フリーの配布を認める、 まっとうなコピーライト条件を持っています。これは、個人的に であっても商業的な目的であっても、OpenBSD のほとんどの部分を 再利用できるということです。OpenBSD は、もともとの BSD ライセンスで含意されているもの以上の制限を一切加えていません。 もっと厳しいライセンス条件のもとで書かれたソフトは、 OpenBSD の正規ディストリビューションには含められません。 これは、OpenBSD のフリーな利用を守るためのものです。 たとえば、OpenBSD は個人利用でも、アカデミックな利用でも、 政府機関でも非営利組織でも、商業組織でも自由に使うことができます。 OpenBSD やその構成部品はまた、自由に 商用製品に利用できます。

我々のフリーな仕事が商用製品 に組み込まれたら困りませんかと時々聞かれます。 答えは、商用製品ベンダーが、再実装し、悪いコードを作りだし、 既に解決済みの問題に対する互換性のない解決をするよりも、 我々のよいコードが 広く使われるほうがいいです。 例えば、SSHはこの自由の ために広く使われるプロトコルですが、OpenSSH のコードに制限が加えられていたら これだけ広く使われる事はなかったでしょう。もしベンダーが自由に使えるSSHソリューションが なかったら（彼らの早急に開発された製品に)、 彼らは書くか すぐに手に入る見窄らしいものを 買うかするでしょう。

これは、我々が金銭的あるいはハードウェアサポート に対する感謝に反対すると言っているのではありません。 実際、呆然とします。OpenBSD(もしくはOpenSSH)に依存した製品を 作っている会社からのどんなとても少しのサポートでも素晴らしいと思います。 ですが、報酬は必須ではありません。

他の一般的なライセンスについてのより詳しいことは、 OpenBSD の著作権のポリシーを参照してください。

OpenBSD のメンテナたちは、プロジェクトをもっぱら自分の持ち出しで サポートしています。これはプロジェクトのためのプログラミングにかけた時間、 各種の移植をサポートするための機器、OpenBSD をあなたに届けるために使われた ネットワーク資源、そしてユーザからの質問に答えたり、バグ報告を調べたり するのに使う時間なんかも含んでいます。しかし、OpenBSD 開発者たち個人は お金持ちではありませんから、ほんの少しでも時間や機器やリソースで貢献して いただければ、大きな違いとなって表われてきます。

1.5 - OpenBSD をサポートするにはどうすれば良いでしょう ?

私たちは、OpenBSD プロジェクトに貢献してきた人々や組織に大いなる 恩があります。彼らのお名前を 寄付のページに掲載しています。

OpenBSD は、ユーザコミュニティからの、常に何通りかのサポートを必要としています。 もし OpenBSD が使えると思ったなら、ぜひともプロジェクトに貢献してください。 もし以下の提案がどれもあなたにとって適切でないのでしたら、 メールで別の方法を提案していただければ幸いです。送り先は donations@openbsd.org です。

  • OpenBSD CD セットの購入。今の OpenBSD のフルリリースが入っていますし、多くのプラットホームでは起動 CD として使えます。 それに、OpenBSD を支える収入を生み出してくれますし、ディストリビューションを インターネット経由で配布するための、ネットワーク資源を節約してくれます。 この、廉価な 3 枚組の CD セットには、すべてのソースコードも含まれています。 そうそう、お友だちのためにも CD を買っておいてあげましょう !
  • お金の寄付。プロジェクトは、 機器やネットワーク接続、CD を発行するための費用など、いつも現金を 必要としているのです。CD のプレスは、OpenBSD 開発者にとって前払いの 持ち出し投資を必要とするもので、しかもそれが帰ってくる保証はありません。 寄付の方法についての詳しいことは、 donations@openbsd.org までメールをお送りください。 ほんの少しのお金でも、大きな差が出るものなのです。
  • 機器やパーツの寄付。プロジェクトは絶えず 一般的なハードや、特定のハードを欲しいと思っています。IDE や SCSI ディスク、 各種 RAM はいつでも大歓迎しています。その他、各種コンピュータシステムや マザーボードの場合、いま何が必要とされているかについては、 まずお問い合わせください。発送のための手続き等については、 donations@openbsd.org まで御連絡ください。
  • 時間や技能で貢献。オペレーティングシステムを書くのが 大好きなプログラマはもちろん大歓迎ですが、 役にたつ方法はそれ以外にも何十とあります。 メーリングリストを読んで、 新しいユーザの質問に答えてあげてください。
  • FAQ のネタを送って、ドキュメンテーションの充実に貢献してください (送り先は faq@openbsd.org です)。 地域のユーザグループを組織して、友だちを OpenBSD の虜にしてしまいましょう。雇用者に、仕事で OpenBSD を使うよう 説得しましょう。学生なら、コンピュータ科学やエンジニアリングの講義で OpenBSD を使いませんか、と教授に話してみましょう。 また、OpenBSD プロジェクトを助けようとして、オペレーティングシステムの フレーム合戦に加わって時間を無駄にしたりしないことが、最も重要なことの ひとつであると意識することも、価値のあることです。 プロジェクトの新しいユーザ探しにも役に立ちませんし、 開発者同士の関係にも非常に有害な影響を与える場合さえあるのです。

1.6 - OpenBSD は誰がメンテナンスしているのでしょう ?

OpenBSD は、世界各国に分散した開発チーム によって維持されています。また、プロジェクトをとりまとめているのは、 カナダ在住の Theo de Raadt です。

1.7 - OpenBSD の次のリリースはいつでしょうか ?

OpenBSD チームでは、5 月 1 日と 11 月 1 日のリリースを目標に、新しい リリースを 6 ヶ月ごとに行っています。開発サイクルに関するより詳しい情報は、 here を参照してください。

1.8 - OpenBSD には何が含まれているのでしょう ?

OpenBSD は、以下のものを含む、多数のサードパーティソフトウェア製品とともに 出荷されています。
  • X.org 7.6, X Window 環境。ローカルパッチ適用済み。 インストール用ファイルセット. x*.tgz がインストールされています。
  • GNU C コンパイラGCC バージョン 2.95.3 と 3.3.5. そして4.2.1 (プラットフォームに依存) OpenBSD チームはスタック保護技術 Propolice を追加し、デフォルトで有効にしています。 OpenBSD ユーザーランドのすべてと OpenBSD で コンパイルされたアプリケーションは、デフォルトでこれが使用されています。 インストール用ファイルセット comp49.tgz の一部としてインストールされています。
  • Perl 5.12.2 （＋パッチ） および OpenBSD チームによる改良。
  • 我々の改良とセキュア化がなされた Apache 1.3 web サーバ。 OpenBSD チームは、デフォルトでの chroot、特権分離、およびその他のセキュリティ関連の改良を 追加しています。また、mod_ssl 2.8 と DSO のサポートも組み込まれています。
  • OpenSSL 1.0.0a（＋パッチ）および OpenBSD チームによる改良。
  • Sendmail 8.14.3 メールサーバ と libmilter メールフィルタライブラリ。
  • BIND 9.4.2-P2 DNS サーバ （＋パッチ）。 OpenBSD では、chroot による運用や他のセキュリティ関連の問題等に対する 多数の改良を実装しています。
  • テキストベース web ブラウザ Lynx 2.8.6rel.5。 また、OpenBSD チームによるパッチと HTTPS サポートとIPv6サポートが追加されています。
  • Sudo v1.7.2p8。これは、ユーザが 個々のコマンドを root として実行することを許可できるようにするためのものです。
  • Ncurses 5.7
  • KAME IPv6
  • Heimdal 0.7.2 （＋パッチ）
  • Arla 0.35.7
  • Binutils 2.15 (＋パッチ)
  • gdb 6.3 (＋パッチ)
  • OpenSSH 5.8
  • OpenNTPD OpenNTPD。セキュアで シンプルなネットワークタイムプロトコル実装。
  • OpenBGPD and OpenOSPFD ルーティングアプリケーション
上記のように、OpenBSD チームでは、(通常は) セキュリティやコードの品質の改良のため、 サードパーティの製品にしばしばパッチを当てることがあります。いくつかのケースでは、 ユーザにとって、操作性の違いを感ずることはありませんが、他の場合では、 操作性の違いが一部のユーザに対して影響を及ぼすことがあるかも知れません。 同じソフトウェアの異なるバージョンをを盲目的に追加する前に、これらの拡張の維持を 心がけてください。 新しいバージョンのものを入手することはできますが、セキュリティの低いシステム になってしまうかもしれません。

もちろん、OpenBSD では、さらなるアプリケーションを packages や ports システムを使用して追加することができます。

1.9 - OpenBSD 4.9 では何が新しくなったのでしょうか ?

4.8から4.9 への変更点の完全なリスト はplus49.htmlに、またハイライトは OpenBSD 4.9 情報にありますが、 ここでは以前のバージョンを熟知しOpenBSD4.9にアップグレードまたは これをインストールしようとする人たちに対し、OpenBSDチームが予想するいくつかの変更は、 以下のように特別の注意が必要となっています。
  • NTFS (読み込みのみ)
    i386 と amd64 の GENERICカーネルで有効になりました。
  • 無線の改善
    以下の簡単なhostname.if フォーマットでWPAキーが指定できるようになりました： 
    nwid myssid wpakey wpapassword
  • 新しい rc.d(8) 機構
    が、従来の/etc/rc.localの編集に代わり、あるいは加えて package デーモンの起動、停止そして再設定のために 使われます。

1.10 - OpenBSDをデスクトップとして使用できますか？

これは、「デスクトップ」という言葉の意味をきちんと説明せずに 尋ねられることが多い質問です。 この質問は「何が必要なのか」に依るため、 答えることができるのは質問した人だけです。

OpenBSDは「サーバ用途の」オペレーティングシステムとして高い評価を得ていますが、デスク トップ用途に使うことも可能です。 多くの" デスクトップ" アプリケーションが packages と portsにあります。 そして、オペレーティングシステムを選ぶときに考えたこと、つまり 「それは自分のやりたい内容を、自分の望むようにやってくれるだろうか」 という質問を考えてください。この質問には、あなたが自分自身で答えを出さなければなりません。

OpenBSDの開発の大きな部分はラップトップ上で行われていることから、 その質問には意味がないかもしれません。

1.11 - どうして あのソフトウェア が入っている/いないのでしょうか？

何故特定の製品がOpenBSDに入っているのか、あるいは入ってないのかと よく聞かれます。 開発者の希望と、我々のプロジェクトの 目標と合致するか、 の二つに基づいている、というのが回答です。 単に性能がよいというだけの理由で、その製品がOpenBSDに含まれる事はありません。ーーその製品が 自由に利用可能であり、我々の標準に合致するよう改良し再配布可能でなければなりません。 また、そのソフトウェアが安定していてセキュアであることも必要ですーーバージョン番号が大きい からといって以前より改良されているとは限らないのです。

よく、大きな問題になるのはライセンスです。わたしたちはOpenBSDを世界中の誰もが、 どんな目的にも使えるようにしたいと考えています。

他に考慮しなくてはいけない事は、開発者の希望です。 OpenBSD 開発者は、何をプロジェクトに追加し、追加しないか、 究極の判断をします。 単にアプリケーションがよいから、というのは OpenBSDプロジェクトがメンテナンスのためのリソースを捧げる事を望んだり、 OpenBSDに 置きつづける他の人の熱意を分け与えることを意味しません。

以下は、いくつかのサードパーティ製ソフトウェアに関してよく尋ねられる質問です。

    • どうして Sendmailが含まれているのでしょうか？あれはセキュアでない事では有名ではないですか？
      いままでSendmail は、 セキュリティに関してよくない経歴を持っていましたが、Sendmail の作者と メンテナはコードをよりセキュアにするための作業に非常に高い理解を 示してくれています（これは滅多にないことです）。 最近のSendmail のセキュリティに関する経歴は、他の「よりセキュアである」とされているソフト ウェアと、ほとんど変わりません。
    • どうしてPostfixが含まれていないのでしょうか？
      ライセンスがフリーではありませんので、 検討される事はありません。
    • どうしてqmail やdjbdnsが含まれてないのですか？
      それらのプログラムはなんと多くのUnixユーザーから消えてほしいと願われているでしょうか。
    • どうして Apache 我含まれているのですか？ほとんどの人には必要ないじゃないで すか！
      開発者が含めることを望んでいるからです。
    • どうして新しいバージョンのApacheは含まれていないのですか？
      新しいバージョンのライセンスは、 受け入れることができないからです。
    • どうしてgzipの代わりにbzip2 が含まれていないのですか？
      パフォーマンスが悲惨で、利が少ないからです。 hp300 や VAX のような遅いプラットフォームには受け入れられません。
    • どうしてグラフィカルな、あるいは curses(3) ベースのインストーラがないのですか？
      いくつかの理由により、インストールメディアを 一枚のフロッピーに納める目標 実際、一つのインストーラはすべての構成、すべてのプラットフォームで使うことができます。 事実として、実際にほとんどのユーザーは2回目か３回目にOpenBSDをインストールした後 OpenBSDのインストーラはもっとも簡単でももっとも速い事がわかるでしょう。 ほとんどの開発者とユーザーは、スピードとパワーと簡単に使える 今のインストーラが、 他のよりカラフルでかわいいインストーラよりも大好きです。
    • ZFSはOpenBSDに追加されますか？
      誰かがOracle にOpenBSD の ポリシー と互換性のあるライセンスに変更させる事を納得させない限り（ないでしょう）。
    これらの話題の多くは、 メーリングリストで 嫌になるほど議論されたものです。もし情報が必要ならば、これらのアーカイブをご覧く ださい。

    もちろん、このようなパッケージを使いたいと望んでおり、ライセンスに違反しない 使い方をしている限り、誰もあなたがそれを使うのを妨げるものはありません （私たちがそうしてしまうと、真にフリーではなくなってしまうのです）。ただし、あなた が必要としているものは変わるかもしれません。つまりあなたはフリーでないソフトウェ アを組み込むことによって、販売や再配布、利益を得ることを禁じるような「キラーアプ リケーション」を開発してしまう化もしれないわけです。そういうのはやめた方がいいで しょう。
