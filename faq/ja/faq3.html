3.1 - OpenBSD CD セットの購入

通常は OpenBSD CD を購入することからはじめるのが適当でしょう。 CD は、 OpenBSD の注文ページから購入することができます。

OpenBSD CD を購入することには、次のような多くの利点があります。

  • CD の売上は、現在行われている OpenBSD の開発資金として利用されます。
  • マルチプラットフォーム対応のオペレーティングシステムの開発には、 開発機器への定期的な投資が必要です。 n
  • あなたの CD を購入は、 将来の開発を支えるのです。
  • CD には、もっとも代表的なサポート済みプラットフォーム向けの バイナリ (とソースコード)がついてきます。
  • CD は、いくつかのプラットフォームで起動メディアとして利用することができます。そのため、予め オペレーティングシステムがインストールされていないマシンの起動に利用することが可能です。
  • CD はスナップショット版をインストールする際の 土台として使うことができます。
  • CD からのインストールは高速です! また、CD からのインストールは、 ネットワークの帯域も節約してくれます。
  • OpenBSD CD には素敵なステッカーがついてきます。これなくしては 完成されたシステムとは言えません。このステッカーは、CD セットを購入するか、 ハードウェアを寄贈しなければ手に入りません。
  • OpenBSD CD セットには 便利で有名な packagesの詰め合わせが入っています。 CD セットはネットワーク接続なしに完全に動かせるようにできますし、 開発環境もつくれます。

リリース版の OpenBSD をインストールするなら、 公式 CD セットを使いましょう。

3.3 - OpenBSD の ISO イメージはダウンロード可能でしょうか ?

プラットフォームを選べば、提供しています。

ほとんどのプラットフォームでは起動可能な OpenBSD インストール CD-ROM を 作成できるISOイメージがダウンロードで入手出来る CD-ROM から 起動する事ができます。 これらのプラットフォームには、"full install" (　OpenBSD のフルインストールに 必要なファイルが入った install48.iso と名付けられた) と "boot only"(ネットワーク 経由でファイルやソースコードを入手する cd48.iso と名付けられた) が提供されています。

このISOイメージファイルは 公式CDセットと同じ ではない ことに注意してください。q これらのイメージは、単一のプラットフォーム向けで、 コンパイル済みパッケージや、ステッカー、アートワーク といった公式 CD セットに含まれているものは入っていません。

しかしながら、ISOファイルでのインストールは多くの人々にとって 最適なインストール方法では”ありません”。 どんなに起動メディアをダウンロードして、必要な一部をインストールすることはたいていは 早くて簡単でも、 しかし、たくさんのインストールを行いたいと願う場合や、十のファイルを CD-ROM やローカル FTP サーバに置く方法が考えられない場合 のために、ISO イメージを利用する事ができます。

OpenBSD プロジェクトでは、公式 CD のマスタとして使用された ISO イメージを ダウンロードして利用できるようにはしていません。この理由は簡単で、 あなたに CD セットを購入していただき、OpenBSD の開発を支援して欲しいと 考えているからです。公式の OpenBSD CD-ROM のレイアウトは Theo de Raadt 氏が 著作権を所有しており、Theo 氏は公式 CD のイメージの再配布を禁止しています。 CD セットを購入してくれる人へのサービスの一つとして、いくつかの特典 (アートワークやステッカーなど) もついています。

ただし、著作権の対象となっているのは CD レイアウトのみであり、 OpenBSD そのものはフリーであることに注意してください。どなたでも OpenBSD をダウンロードして、独自に CD を作成しても一向に構いません。

起動可能な CD を必要とする人たちのために、起動ディスクの ISO イメージ (名称は cd48.iso) が、システムの残りの部分を FTP 経由でインストール することができる、多くのプラットフォーム向けに用意されています。これらの ISO イメージは数メガバイト程度のサイズしかなく、実際のファイル一式ではなくて、 インストールに必要なツールだけを含むものです。

3.4 - FTP, HTTP, AFS 経由でのダウンロード

3.5 - ハードウェアの選択

あなたの OpenBSD システムを動かすのにふさわしいハードウェアの選択は とても重要です。プロジェクトの正否を意味するようなものです。

もしあなたが新しい PC を買いに行ったなら、 パーツを一つ一つ買うにせよ完成品を買うにせよ、最初に望む事は 信頼出来るパーツを買いたいということでしょう。 PC の世界では、これは簡単ではありません。 悪いパーツ、あるいは信頼出来ない、もしくは合わないパーツは OpenBSD を不完全に動かし、しばしばクラッシュさせます。 私たちが与えられる最良の助言は、気をつけて、そしてあなたが信頼する権威がレビューした ブランドやパーツを買うことです。 時に、高い値段のマシンはより良い質のマシンです。 別の時には、それは単に高いだけです。

確かな事は、システムの最大のパフォーマンスを発揮させます。

  • アプリケーションがハードウェアを選ばせます : It is usually better to make an adjustment to the hardware you were planning on using rather than compromising on your application design because you have something you "really wanted to use".
  • ボトルネックを特定しましょう アプリケーションがディスク I/O に制限されているときに 追加で最新のプロセッサーにお金を費やさないでください。 システムがネットワーク速度に制限されているときに速いディスクを買わないでください。 あなたのボトルネックが 128kbps の DSL にあるときに他のものを買わないでください。
  • 簡潔さを保ちましょう : シンプルなハードウェアはたいていシンプルな問題を起こします。 複雑なハードウェアは壊れる事を想定していないので一度壊れると 修理に多くの時間がかかります。
  • あなたが理解しているハードウェアを使いましょう。あるいは製品システム を実装する前に新しいハードウェアについて学習しましょう: ハードウェアの技術的利点に注意しない、 あなたが慣れ親しむ前に特別なハードウェアを使う それらがどう動いているのか、そしてどう失敗しているのか の両方、それは愚かなことです。
  • 複数のディスクを使いましょう : ひとつの大きなディスクを買う代わりに、複数のより小さなディスクを買いましょう。これは 高くつくかもしれませんが、複数のスピンドルに負荷を分散させることは、ディスク上のデータ にアクセスするのに必要な時間を少なくする事ができます。そして、多くの スピンドルがあれば、RAID による高信頼性と高速なデータアクセスをてに入れる事ができます。
  • ストレージの大きなブロックを分割しましょう 多くの人々は大量のデータをストレージの一つの大きなブロックに保存する ようにシステムをデザインするという過ちを犯しています。 これは普通は悪いデザインです。 あなたはいつもそれをみつけたらストレージをより管理しやすいブロックに 分割するべきです。 これには多くの利点があります、ふたつのもっとも大きなものは、あなたは 後で必要になったときに簡単に追加することができます (あなたは 必要な正しい見積もりを得ることはできないでしょう、誰にもできません) 。そしてあなたは 少量のストレージを今買う事ができます、そしてさらに後で追加するときには 価格が下がり容量は大きくなっていることでしょう。
  • 安いネットワークアダプタは避けましょう : OpenBSD は大量の安価なネットワークアダプタをサポートしています。 これらのアダプタは家では素晴らしくよく動きますが、ビジネスや研究の環境では 低いか並のスループットしか出せません。 しかし、サーバに高いスループットと低い衝撃が必要であれば、あなたは 質のよいネットワークアダプタを買う方が懸命でしょう。 不運にも、いくつかの名のあるアダプタは 安いアダプタよりもよくないです、そして良いアダプタの可能性のあるものは よいドライバを書くための詳細なドキュメントがありません。 ギガビットアダプタは遅いネットワークで使用した時でも優れたバッファのために しばしば 10Mbps/100Mbps のアダプタよりもよいパフォーマンスを発揮します。

3.6 - OpenBSD を学習するのに最適な"最初のシステム"はなんでしょうか ?

OpenBSD は他のモダンなOSに比べて小さく、古く、パワーのないコンピュータ で動くでしょうけれども、そのようなあまりにも小さなマシンで OpenBSD をはじめると がっかりするでしょう。 以下のガイドラインは単にガイドラインです。 OpenBSD は ここに挙げられているよりもしょぼい設備でも動きますが、はじめてのユーザーに とってよりがっかりさせられるでしょう。
  • プラットフォーム あなたが既に慣れ親しんだプラットフォームを使いましょう。 新しいオペレーティングシステムを学ぶ時は、 新しいプラットフォームを学ぶには悪い時です。 我々はあなたが i386 プラットフォームを使うと決めて かかっています。多くの人々が慣れ親しんでいるでしょうから。
  • プロセッサ 100MHz の Pentium かそれ以上のプロセッサ。 もちろん、 OpenBSD は 25MHz の 80486 でも動くでしょう、しかし 実験し、無茶苦茶にしてリロードして、本当にそのシステムが遅いマシンであると 知る事をあなたは欲しないでしょう。 プロセッサが遅いことによる最初に遭遇する第一のいらだちは SSH でログインする際に時間がかかることです。 もし X を動かしたいのであれば、少なくとも 200MHz 以上が必要になるでしょう。 X は実は一度ロードされてしまえば遅いマシンでも軽快に動きます、しかし ロードと始動にしばらく時間がかかります。
  • 64M 以上のメモリ もし X を動かしたいのであれば、 128MB を開始点にするのがよいでしょう。
  • ハードディスク ファイヤーウォールやDNSサーバのような単純なシステムを簡単にインストール するのであれば、1G のハードディスクがあればよいでしょう。 もしシステムをソースから再構築したいのであれば、多分 4G は 欲しいところですし、 X を同様に再構築したいのであれば、 6G 以上は欲しいでしょう、 始めるにあたっては、IDE を推奨します。 もしもっと大きなディスクを持っていても、全てを一度に確保する義理を 感じないでください -- 全てに必要なのが 8G だったときに 80G のディスクのうち 72G を確保しない事は何も悪い事はありません。
  • ネットワークアダプタ PCI のアダプタを使ってください。 古い ISAアダプターを使う事を進められたら、強く抵抗するべきです。 あなたはおそらくちゃんと設定する方法を覚えていないでしょう。 もし、マシンに複数のネットワークアダプターをつけることを計画しているのであれば カバーを取り付ける前に、カードの背に MAC アドレスを 書いておきましょう。
  • マルチブート 始めて OpenBSD をインストールするのであれば、他の OS とのマルチブートを 試みないでください。 マルチブート は正しく動作させるには 難しい手順ですし、マルチブートを試みる前に関係する全ての OS を理解していなければ なりませんし、明らかに最初のインストールで 試みることではありません。 誤ってシステムの全てのデータを消す事は十分可能です。 むしろ、専用のコンピュータを使いましょう、少なくともマシンに専用のディスクを用意するべきです。
  • ノートパソコン 多くのノートパソコンで OpenBSD はとても快適に動作しますが、快適に動作させるのは 簡単なッステムではない場合が時々あります、ですのでノートパソコン を最初の OpenBSD インストールに用いるのは最良の選択ではありません。 しかしながら、一度 OpenBSD を楽に操れるようになると、ノートパソコンはとても便利 なツールになります。
  • 新しいハードウェア 新しい、最先端のハードウェアはOpenBSDでまだサポートされていないときが あります。ですので、最初の OpenBSD システムには、少し古いマシンが 推奨されます。
"よりベターな" がポイントなのは明白です。 一見したところ、いくつかの有名なアプリケーションは、システムに投入しうる もっと速いプロセッサとメモリを使うことができます。
